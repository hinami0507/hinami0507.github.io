小红书-(11)-基本包装类型

By hinami

发表于 2015-10-28

ECMAscript提供了3个特殊的引用类型:

  • Boolean
  • Number
  • String

他们具备各自基本类型的特殊行为

substring()方法
上边的代码中s1是基本类型，然而在第二行中，却调用了其substring()方法，这看起来好像是把他当引用类型来使唤了,但是这是不同的！

基本类型和引用类型的不同在于其对象的生存期

在第二行给s1赋color属性，但是在第三行却找不到了，因为第二行运行完后，这个属性就被扔掉了，所以不能给基本类型添加属性和方法

string的一大波方法！

  • charAt(),charCodeAt(),stringValue[]查找该位置的字符

concat拼接字符串

slice()、substring()、substr()截取字符串(创建新字符串)

indexOf()、lastIndexOf()字符串位置方法

这里之前有讲过，这里就不讲了

trim()删除字符串前后空格

支持这个方法的浏览器有IE9+、Firefox 3.5+、Safari 5+、Opera 10.5+和Chrome

字符串大小写转换方法

toLowerCase()、toLocaleLowerCase()、toUpperCase()和toLocaleUpperCase()
在不知道自己的代码将在哪种语言环境中运行的情况下，还是使用针对地区(用local)的方法更稳妥一些。

字符串的模式匹配方法

  • 查找字符
    match()和exec()几乎相同，只是语句有点不同
    match()只接受一个参数,RegExp或者是一个正则表达式
  • 替换字符

replace接受两个参数

  • 第一个参数：RegExp(推荐)或者一个字符串
  • 第二个参数：字符串或者函数

替换并且返回被替换的字符串

然而replace真正强大之处在于可以放入函数！
可以传入三个参数:

  • 模式的匹配项
  • 模式匹配项在字符串中的位置
  • 原始字符串
  • split()(分裂)，将字符串切碎放入数组中
    两个参数
  • 第一个参数：RegExp
  • 第二个参数(可选)：数值，规定返回的数组长短

字符串排序

localeCompare()

ASCII码转为字符

fromCharCode()

最强大的eval()

当解析器发现代码中调用eval()方法时，它会将传入的参数当作实际的ECMAScript 语句来解析，然后把执行结果插入到原位置通过eval()执行的代码被认为是包含该次调用的执行环境的一部分，因此被执行的代码具有与该执行环境相同的作用域链。

就像这句话eval("alert('hi')");相当于alert('hi')
